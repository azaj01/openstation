# Cli Create --Body Flag — Accept Task Body Via Flag Or Stdin For One-Shot Task

## Requirements

Add a `--body` option to `openstation create` so a task can be
created with full requirements and verification in one command.

### Interface

```bash
# Via flag (inline)
openstation create "description" --body "## Requirements\n..."

# Via file
openstation create "description" --body-file path/to/body.md

# Via stdin
echo "## Requirements..." | openstation create "description" --body -
```

### Behavior

- Body content replaces the skeleton `## Requirements` /
  `## Verification` sections that the CLI currently generates
- Body is markdown (everything after frontmatter)
- If `--body` is not provided, current behavior is unchanged
  (skeleton with placeholder)
- `--body-file` reads from a file path
- `--body -` reads from stdin

### Update `/openstation.create` Command

Update the slash command procedure to use `--body` when creating
tasks, eliminating the two-step create-then-edit flow (steps 4+5
become one step).

## Findings

Implemented `--body` and `--body-file` flags for `openstation create`:

**Code changes:**
- `src/openstation/cli.py` — Added `--body` and `--body-file` as mutually exclusive arguments to the create subparser. Updated help epilog with examples.
- `src/openstation/tasks.py` — Added `_resolve_body()` helper that resolves body content from `--body` (inline or stdin via `-`) or `--body-file` (file path). `cmd_create()` uses the resolved body instead of the hardcoded skeleton when provided.
- `.openstation/commands/openstation.create.md` — Merged steps 4+5 into a single step using `--body` flag. Eliminated the two-step create-then-edit flow. Updated sub-task creation similarly.
- `.openstation/docs/cli.md` — Added `--body` and `--body-file` to synopsis, flags table, and examples.

**Design decisions:**
- `--body` and `--body-file` are mutually exclusive (argparse group)
- `--body -` reads stdin; errors if stdin is a TTY (no piped input)
- Body replaces everything after the H1 title — frontmatter and title are always generated by the CLI
- Empty/missing `--body` preserves the existing skeleton behavior

**Tests:** 7 new tests in `tests/test_cli.py` covering inline body, body-file, stdin, skeleton preservation, frontmatter integrity, missing file error, and mutual exclusivity. All 275 CLI tests pass.
